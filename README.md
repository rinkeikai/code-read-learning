# code-read-learning

**AI Agent による実装が当たり前になった開発環境において、コード読解力（Code Reading Skill）を維持・向上させるための MCP サーバーです。**

AI Agent はコード生成を高速化します。

しかし、多くの場合エンジニアは、

* 「何の機能を実装したか」は理解している
* 「なぜ実装したか」も理解している

一方で、

* この変数は何を保持しているのか
* この条件分岐は何を判定しているのか
* この演算子は何を意味するのか
* この関数はどこから呼ばれているのか

といったコードレベルの説明ができないまま実装を進めてしまいます。

これは AI の性能向上によって、

**「コードを書く速度」と「コードを読む速度」の差が拡大していること**が原因です。

---

## 解決したい課題

例えば以下のコードがあります。

```js
const type = resolved.layoutType ?? params.type;
```

多くの開発者は、

「layoutType があればそれを使う処理」

というレベルでは理解できます。

しかしレビューで、

> この1行は何をしていますか？

と聞かれた場合、

* `const`
* プロパティアクセス
* Nullish Coalescing Operator (`??`)
* 評価順序
* 代入される値の決定条件

まで説明できるとは限りません。

また、AI Agent による実装では、

* 実装内容は理解している
* 設計意図も理解している

にもかかわらず、

* コードを1行ずつ読めない
* 処理を日本語に翻訳できない
* レビューで詳細を説明できない

という状態が発生しやすくなります。

---

code-read-learning は、

**AI が生成したコードを教材として利用し、コード読解力そのものを鍛えること**

を目的としています。

---

## コンセプト

一般的な学習教材は、

* チュートリアル
* サンプルコード
* アルゴリズム問題

を扱います。

しかし実際の業務で読むコードは、

昨日 AI が生成したプロダクトコードです。

そのため code-read-learning では、

学習対象として

* `git diff --staged`
* `git show HEAD`

から取得した

**自分自身の実装差分**

を利用します。

つまり、

**今日実装したコードが今日の教材になります。**

---

## 何を目指すのか

このツールの目的は、

コードを理解した気になることではありません。

レビューで、

> この1行は何をしていますか？

と聞かれた時に、

自分の言葉で説明できる状態を目指します。

例えば、

```js
const type = resolved.layoutType ?? params.type;
```

を見て、

「layoutType があればそれを使う」

で終わるのではなく、

```text
const による定数宣言。

resolved オブジェクトの layoutType プロパティを取得する。

layoutType が null または undefined の場合のみ、
params.type を利用する。

その結果を type に代入する。
```

というレベルで説明できる状態を目標とします。

## このツールの役割

このツールはコード解説ツールではありません。

また、コードレビューを自動化するツールでもありません。

### 提供する機能

* Git差分取得
* 変更ファイル抽出
* 変更関数抽出
* 読解順生成
* 学習セッション用プロンプト生成

### 提供しない機能

* コード要約
* 設計解説
* アーキテクチャ解説
* 学習の自動化
* コード理解の代行

学習そのものは Cursor AI が担当します。

code-read-learning は、

**読解セッションを開始するための教材生成基盤**

として機能します。

---

## 主な機能

### `list_modules`

学習対象となる Git リポジトリおよびサブモジュールを列挙します。

複数モジュール構成のプロジェクトでも、読解対象を明確に選択できます。

---

### `get_learning_material`

Git差分から読解教材を生成します。

取得優先順位：

1. `git diff --staged`
2. `git show HEAD`

返却内容：

```json
{
  "files": [],
  "functions": [],
  "diff": "",
  "recommendedOrder": [],
  "learningPrompt": ""
}
```

## Philosophy

AI によって、

コードを書くコストは大きく下がりました。

しかし、

コードを読む責任は依然としてエンジニアにあります。

AI が生成したコードを、

理解したつもりで次の実装へ進まない。

コードを日本語へ翻訳できるまで読む。

code-read-learning は、

AI 時代のエンジニアのための

**Code Reading Training Environment**

です。
